Mix.Tasks.Compile.App

Writes an .app file.

By default, this task will detect all modules in your compile_path (defaults to ebin) and generate a best guess for your application specification. This best guess also includes kernel, stdlib and elixir as application dependencies.

You can optionally define an application/0 function inside your Mix.Project that returns a keyword list to further configure your application according to the OTP design principles:

http://www.erlang.org/doc/design_principles/applications.html

Command line options

  • --force - forces compilation regardless of modification times

Configuration

  • :app - The application name as a binary (required)
  • :version - The application version as a binary (required)
