Ecto v0.0.1

Modules summary

Ecto.Adapter

This module specifies the adapter API that an adapter is required to implement.
Ecto.Adapter.Migrations

This module specifies the migrations API that an adapter is required to implement.
Ecto.Associations

Utilities on associations.
Ecto.Associations.BelongsTo

A belongs_to association.
Ecto.Associations.HasMany

A has_many association.
Ecto.Associations.HasOne

A has_one association.
Ecto.Entity

This module is used to define an entity. An entity is a record with associated meta data that is persisted to a repository.
Ecto.Migration
Ecto.Migrator

This module provides the migration API.
Ecto.Model

Models are Elixir modules with Ecto-specific behaviour.
Ecto.Model.Queryable

Defines a model as queryable.
Ecto.Model.Validations

Conveniences for defining module-level validations in models.
Ecto.Query

This module is the query DSL. Queries are used to fetch data from a repository (see Ecto.Repo).
Ecto.Query.API
Ecto.Query.Util

This module provide utility functions on queries.
Ecto.Repo

This module is used to define a repository. A repository maps to a data store, for example an SQL database. A repository must implement url/0 and set an adapter (see Ecto.Adapter) to be used for the repository.
Ecto.Utils

Convenience functions used throughout Ecto and imported into users modules.
Ecto.Validator

Validates a given record or dict given a set of predicates.
Ecto.Validator.Predicates

A handful of predicates to be used in validations.
Mix.Tasks.Ecto.Gen.Migration

Generates a migration for the given repository.
Mix.Tasks.Ecto.Gen.Repo

Generates a new repository.
Mix.Tasks.Ecto.Migrate

Runs the pending migrations for the given repository. Migrations are expected to be found inside the migrations directory returned by the priv function defined in the repository.

Protocols summary

Ecto.Queryable

The Queryable protocol is responsible for converting a structure to an Ecto.Query.Query record. The only function required to implement is to_query which does the conversion.
